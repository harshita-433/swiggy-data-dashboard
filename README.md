
🍔 Swiggy Order Analytics Dashboard

📌 Project Overview
The Swiggy Order Analytics Dashboard is an interactive Microsoft Excel dashboard developed to analyze customer food ordering patterns, spending behavior, restaurant performance, payment methods, dietary preferences, and coupon usage. It transforms raw transactional data into meaningful business insights using **Pivot Tables, Pivot Charts, Slicers, KPI Cards, and Excel formulas**.

This project demonstrates how Microsoft Excel can be effectively used as a Business Intelligence (BI) tool for data cleaning, analysis, visualization, and interactive reporting.

🎯 Project Objective

The primary objective of this project is to analyze Swiggy order data and answer key business questions that support data-driven decision-making, including:

- Which restaurants receive the highest number of orders?
- How much do customers spend over time?
- Which payment method is used the most?
- Which coupons are applied most frequently?
- What are customers' dietary preferences (Veg vs. Non-Veg)?
- During which months are orders highest?
- What trends can help improve business decisions and customer experience?

📂 Dataset Information

The dataset consists of 206 Swiggy order records, capturing various aspects of customer ordering behavior. It includes transaction details, restaurant information, payment methods, customer preferences, and delivery-related attributes. This data was used to perform exploratory analysis and build an interactive Excel dashboard.

🧹 Data Cleaning & Preprocessing

Before building the dashboard, the dataset was cleaned and transformed to ensure data accuracy, consistency, and reliability.

Data Cleaning Steps: 
- Removed duplicate records to eliminate redundancy.
- Converted date and time values into a consistent Excel date format.
- Created an **Order Hour** column for time-based analysis.
- Extracted the **Day Name** from the order date to analyze weekday trends.
- Standardized payment method names for consistency.
- Classified customer orders into **Veg** and **Non-Veg** categories.
- Created a **Discount Applied** column to analyze coupon usage.
- Checked for and handled missing or inconsistent values.
- Structured the dataset for efficient analysis using Pivot Tables and Pivot Charts.
